# Prompts: Market Readiness

## 1. Objetivo del documento

Este documento reúne prompts operativos para apoyar el servicio de Market Readiness de ARNI Consulting.

Los prompts están diseñados para ayudar a ordenar información, detectar brechas, evaluar preparación comercial, identificar riesgos, clasificar readiness y preparar recomendaciones antes de avanzar a Importer Scouting, Pilot Validation, Business Matchmaking u otros servicios ARNI.

La IA puede apoyar el análisis y la redacción, pero toda conclusión debe ser validada por una persona antes de usarse con clientes.

---

## 2. Prompt para diagnóstico inicial de Market Readiness

```text
Actúa como analista de Market Readiness para ARNI Consulting.

Con la siguiente información, prepara un diagnóstico inicial de preparación comercial para entrar o validar un mercado objetivo.

Incluye:

1. Resumen del producto o servicio.
2. Mercado objetivo.
3. Canal comercial considerado.
4. Información disponible.
5. Información faltante.
6. Brechas principales.
7. Riesgos comerciales.
8. Riesgos de precio.
9. Riesgos documentales.
10. Riesgos logísticos o regulatorios preliminares.
11. Preguntas que deben hacerse al cliente.
12. Clasificación preliminar: Ready, Almost Ready o Not Ready.
13. Siguiente paso recomendado.

No inventes datos. Si falta información, indícalo claramente.

Información:
[PEGAR INFORMACIÓN]
```

## 4. Prompt para detectar brechas

```text
Actúa como revisor crítico de preparación comercial internacional.

Identifica las brechas que podrían impedir que esta empresa avance a búsqueda de importadores, distribuidores, compradores o socios comerciales.

Clasifica las brechas en:

1. Producto.
2. Precio.
3. Capacidad.
4. Documentación.
5. Certificaciones.
6. Canal.
7. Mercado.
8. Propuesta de valor.
9. Riesgos regulatorios.
10. Riesgos logísticos.
11. Riesgos comerciales.

Para cada brecha entrega:

- Descripción.
- Nivel de prioridad: alta, media o baja.
- Riesgo si no se corrige.
- Acción recomendada.

Información:
[PEGAR INFORMACIÓN]
```

## 14. Prompt para revisión crítica antes de entregar al cliente

```text
Actúa como revisor crítico de calidad para un diagnóstico de Market Readiness.

Evalúa si el diagnóstico está listo para entregarse al cliente.

Revisa:

1. Si el producto está bien explicado.
2. Si el mercado objetivo está claro.
3. Si la evaluación de precio es prudente.
4. Si la capacidad está suficientemente validada.
5. Si la documentación disponible está bien identificada.
6. Si las brechas están claramente explicadas.
7. Si los riesgos están bien clasificados.
8. Si la clasificación Ready / Almost Ready / Not Ready está justificada.
9. Si hay afirmaciones no verificadas.
10. Si hay promesas comerciales, regulatorias o legales indebidas.
11. Si las recomendaciones son accionables.
12. Qué debe corregirse antes de entregar.

Diagnóstico:
[PEGAR DIAGNÓSTICO]
```

---

## 15. Reglas de uso

1. No usar respuestas de IA como diagnóstico final sin revisión humana.
2. No inventar precios, certificaciones, capacidades, clientes, mercados o requisitos.
3. No prometer ventas, aceptación del mercado, aprobación regulatoria ni éxito comercial.
4. No emitir conclusiones legales, sanitarias, regulatorias o aduanales definitivas sin especialista.
5. Toda clasificación Ready / Almost Ready / Not Ready debe estar justificada con evidencia.
6. Toda recomendación debe distinguir entre hechos, supuestos y riesgos.
7. Si falta información, debe señalarse explícitamente.
8. Si el producto no está listo, no recomendar prospección inmediata.
9. El entregable final debe ayudar al cliente a decidir qué corregir y qué siguiente paso tomar.
