# Roadmap del Manual ARNI Consulting

Este archivo sirve para controlar el avance del Manual ARNI Consulting.

Mantener una visión clara de las secciones creadas, pendientes y prioritarias del manual operativo.

## Reglas de actualización

Cada nueva sección debe cumplir con:

- Título claro.
- Objetivo.
- Cuándo usarla.
- Pasos operativos.
- Entregables.
- Riesgos o límites.
- Principio de uso.

## Control de cambios

Cada cambio debe guardarse con un commit claro.

Ejemplos:

- Agregar playbook de automatización IA.
- Actualizar reglas operativas.
- Agregar plantilla de diagnóstico.
- Corregir índice del README.
- Agregar criterios de calificación comercial.

## Principio del roadmap

El manual debe crecer de forma ordenada. No se trata de acumular documentos, sino de construir una base operativa reutilizable para ARNI Consulting.
