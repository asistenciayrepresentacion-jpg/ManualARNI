# Playbook Automatización e IA Aplicada

Este playbook documenta el proceso base para identificar, diseñar y preparar automatizaciones o agentes de IA aplicados a procesos comerciales, analíticos y operativos de ARNI Consulting.

## Objetivo del servicio

Ayudar a empresas, instituciones o equipos internos a mejorar procesos mediante automatización, agentes de IA, bases estructuradas, prompts operativos y flujos de trabajo supervisados.

El objetivo no es automatizar por moda, sino resolver problemas medibles de tiempo, seguimiento, clasificación, análisis o documentación.

## Cuándo usar este playbook

Este proceso aplica cuando existe una necesidad relacionada con:

- Procesos repetitivos.
- Seguimiento comercial.
- Clasificación de oportunidades.
- Análisis de bases de datos.
- Preparación de reportes.
- Minutas.
- Generación de propuestas.
- Gestión de contactos.
- Priorización de prospectos.
- Integración de herramientas como Airtable, n8n, CRM, formularios o agentes IA.

## 1. Identificación del problema

Antes de proponer automatización, ARNI debe entender el problema operativo.

Preguntas clave:

- ¿Qué proceso consume demasiado tiempo?
- ¿Qué actividad se repite con frecuencia?
- ¿Qué información se pierde?
- ¿Dónde hay errores humanos?
- ¿Qué decisión necesita más claridad?
- ¿Qué seguimiento no se está haciendo bien?
- ¿Qué tarea podría ser asistida por IA sin perder control humano?

## 2. Evaluación del caso de uso

No todo debe automatizarse.

Criterios para evaluar si un caso vale la pena:

- Frecuencia del proceso.
- Volumen de información.
- Costo del error.
- Tiempo consumido.
- Claridad de reglas.
- Disponibilidad de datos.
- Riesgo de automatización.
- Necesidad de supervisión humana.
- Impacto esperado.

## 3. Clasificación del tipo de solución

La solución puede ser:

### Automatización simple

Flujo que conecta herramientas y reduce tareas manuales.

Ejemplos:

- Formulario a base de datos.
- Base de datos a correo.
- Recordatorio automático.
- Registro de reunión.
- Actualización de estado.

### Agente IA supervisado

Sistema que analiza información y sugiere acciones, pero requiere validación humana.

Ejemplos:

- Agente comercial.
- Agente de seguimiento B2B.
- Agente de clasificación de oportunidades.
- Agente de preparación de minutas.
- Agente de generación de propuestas.

### Herramienta interna

Aplicación, calculadora, dashboard o plantilla digital.

Ejemplos:

- Calculadora de landed cost.
- Matriz de priorización.
- Scoring de proveedores.
- Dashboard comercial.
- Base de seguimiento.

## 4. Recolección de información

Para diseñar la solución se debe recopilar:

- Proceso actual.
- Herramientas usadas.
- Responsables.
- Frecuencia.
- Datos disponibles.
- Formatos actuales.
- Problemas principales.
- Resultado esperado.
- Restricciones.
- Nivel de acceso permitido.
- Riesgos de privacidad o confidencialidad.

## 5. Diseño del flujo

Cada flujo debe documentar:

- Entrada de información.
- Procesamiento.
- Criterios de decisión.
- Salida esperada.
- Responsable humano.
- Herramientas involucradas.
- Riesgos.
- Puntos de validación.
- Condiciones para detener el flujo.

## 6. Herramientas posibles

Dependiendo del caso, pueden usarse:

- Airtable.
- n8n.
- GitHub.
- Google Sheets.
- Google Forms.
- CRM.
- ChatGPT.
- Claude.
- APIs.
- Dashboards.
- Formularios web.
- Bases internas.

## 7. Reglas para agentes IA

Todo agente IA debe tener reglas claras:

- Qué puede hacer.
- Qué no puede hacer.
- Qué datos puede modificar.
- Qué datos no puede modificar.
- Cuándo debe pedir confirmación.
- Qué salida debe entregar.
- Qué riesgos debe señalar.
- Cómo debe registrar sus acciones.

## 8. MVP o piloto

Antes de construir una solución completa, ARNI debe proponer un MVP.

El MVP debe:

- Resolver un problema concreto.
- Usar pocos campos críticos.
- Tener bajo riesgo operativo.
- Ser probado con datos reales o simulados.
- Permitir corrección rápida.
- Tener revisión humana.
- Medir utilidad.

## 9. Indicadores de éxito

La solución puede evaluarse con:

- Tiempo ahorrado.
- Reducción de errores.
- Mayor seguimiento.
- Mejor priorización.
- Mejor calidad documental.
- Mayor velocidad de respuesta.
- Mayor trazabilidad.
- Menor dependencia de memoria individual.

## 10. Entregables

El servicio puede entregar:

- Diagnóstico del proceso.
- Mapa de flujo.
- Diseño del agente o automatización.
- Base operativa.
- Prompts.
- Reglas de uso.
- MVP funcional.
- Manual de operación.
- Recomendaciones de mejora.

## 11. Principio del playbook

La automatización debe aumentar control, claridad y capacidad operativa. No debe reemplazar criterio profesional ni crear riesgos por exceso de autonomía.
