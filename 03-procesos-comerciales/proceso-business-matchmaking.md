# Proceso Operativo: Business Matchmaking

## 1. Objetivo del proceso

Este proceso establece los pasos operativos para preparar, coordinar, ejecutar, documentar y dar seguimiento a reuniones B2B entre empresas y contrapartes relevantes en un mercado objetivo.

Su propósito es asegurar que las reuniones tengan objetivos claros, contrapartes con fit razonable, preparación previa, minutas, seguimiento y criterios para distinguir reuniones informativas de oportunidades comerciales reales.

---

## 2. Resultado esperado

Al finalizar el proceso, ARNI debe poder responder:

1. ¿Cuál era el objetivo de cada reunión?
2. ¿La contraparte tenía fit real?
3. ¿Qué se presentó?
4. ¿Qué interés o necesidad se detectó?
5. ¿Qué objeciones o dudas surgieron?
6. ¿Qué información se solicitó?
7. ¿Qué siguiente acción se acordó?
8. ¿La reunión debe convertirse en oportunidad comercial?
9. ¿Quién es responsable del seguimiento?

---

## 3. Insumos necesarios

Antes de iniciar, el cliente o participante debe entregar o confirmar:

* Nombre de la empresa.
* Producto, servicio o solución.
* Mercado objetivo.
* Tipo de contraparte buscada.
* Objetivo comercial.
* Perfil de cliente ideal.
* Pitch o presentación.
* Catálogo, ficha técnica o material comercial.
* Precio o rango de referencia, si aplica.
* Capacidad de producción o entrega.
* Restricciones conocidas.
* Disponibilidad para reuniones.
* Responsable de seguimiento.
* Criterios para considerar una reunión útil.

---

## 4. Etapas del proceso

## Etapa 1: Definir objetivo del matchmaking

### Objetivo

Aclarar qué se quiere lograr con las reuniones B2B.

### Actividades

1. Confirmar el objetivo comercial del cliente.
2. Definir si se buscan compradores, importadores, distribuidores, socios, instituciones o aliados.
3. Identificar qué tipo de reunión se necesita.
4. Aclarar qué información debe obtenerse de cada reunión.
5. Definir qué resultado se considerará útil.
6. Evitar objetivos demasiado amplios o poco medibles.

### Ejemplos de objetivos

* Identificar compradores potenciales.
* Validar interés comercial.
* Encontrar distribuidores.
* Explorar alianzas.
* Preparar agenda B2B para una misión comercial.
* Conectar con instituciones de apoyo.
* Dar seguimiento a contactos previos.
* Convertir reuniones en oportunidades.

### Resultado esperado

Objetivo claro del matchmaking.

---

## Etapa 2: Perfilar al cliente o participante

### Objetivo

Preparar la información mínima para presentar al cliente ante contrapartes.

### Actividades

1. Documentar empresa, sector y país de origen.
2. Describir producto, servicio o solución.
3. Identificar propuesta de valor.
4. Confirmar capacidad comercial u operativa.
5. Revisar materiales disponibles.
6. Confirmar mercado objetivo.
7. Detectar restricciones o temas sensibles.
8. Preparar resumen breve para uso interno.

### Campos sugeridos

| Campo              | Descripción                                        |
| ------------------ | -------------------------------------------------- |
| Empresa            | Nombre del cliente o participante                  |
| Sector             | Industria o categoría                              |
| Producto/servicio  | Oferta principal                                   |
| Mercado objetivo   | País, región o canal buscado                       |
| Propuesta de valor | Razón de interés para la contraparte               |
| Capacidad          | Producción, entrega, cobertura o experiencia       |
| Materiales         | Catálogo, ficha, presentación, sitio web           |
| Objetivo           | Qué busca lograr con reuniones                     |
| Restricciones      | Precio, logística, certificaciones, idioma, tiempo |

### Resultado esperado

Perfil interno del cliente o participante.

---

## Etapa 3: Definir perfil ideal de contraparte

### Objetivo

Determinar qué tipo de empresa, institución o persona debe buscarse.

### Actividades

1. Definir tipo de contraparte.
2. Definir canal prioritario.
3. Definir sector o categoría.
4. Definir región o cobertura.
5. Identificar criterios de buen fit.
6. Identificar criterios de descarte.
7. Priorizar contrapartes según relevancia.

### Tipos de contraparte

* Importador.
* Distribuidor.
* Retailer.
* Broker.
* Food service.
* HORECA.
* Cliente industrial.
* Cámara empresarial.
* Asociación sectorial.
* Agencia de promoción.
* Socio estratégico.
* Consultor especializado.
* Integrador.
* Representante local.

### Resultado esperado

Perfil ideal de contraparte.

---

## Etapa 4: Identificar contrapartes potenciales

### Objetivo

Construir o revisar una base de posibles reuniones.

### Fuentes posibles

* Bases internas de ARNI.
* Importer Scouting.
* Directorios comerciales.
* Cámaras empresariales.
* Asociaciones sectoriales.
* Ferias y eventos.
* Referencias institucionales.
* LinkedIn.
* Sitios web de empresas.
* Contactos previos.
* Recomendaciones de aliados.

### Campos mínimos

| Campo               | Descripción                                                   |
| ------------------- | ------------------------------------------------------------- |
| Empresa/institución | Nombre de la contraparte                                      |
| Tipo                | Comprador, importador, distribuidor, socio, institución, etc. |
| País/ciudad         | Ubicación                                                     |
| Sector              | Industria principal                                           |
| Canal               | Canal en el que opera                                         |
| Contacto            | Persona identificada                                          |
| Cargo               | Puesto                                                        |
| Email/teléfono      | Datos disponibles                                             |
| Fuente              | Origen de la información                                      |
| Razón de fit        | Por qué podría ser relevante                                  |
| Prioridad           | A, B o C                                                      |
| Estado              | Identificada, contactada, confirmada, etc.                    |

### Resultado esperado

Base inicial de contrapartes.

---

## Etapa 5: Calificar y priorizar contrapartes

### Objetivo

Evitar agendas llenas de reuniones sin fit.

### Criterios de priorización

| Criterio           | Pregunta clave                                  |
| ------------------ | ----------------------------------------------- |
| Sector             | ¿Opera en la industria correcta?                |
| Canal              | ¿Pertenece al canal buscado?                    |
| Capacidad          | ¿Tiene señales de operación real?               |
| Relevancia         | ¿La reunión puede generar aprendizaje o avance? |
| Cobertura          | ¿Opera en el mercado objetivo?                  |
| Interés potencial  | ¿Hay razón para que le interese la oferta?      |
| Accesibilidad      | ¿Existe vía razonable de contacto?              |
| Riesgo de bajo fit | ¿Hay señales de pérdida de tiempo?              |

### Clasificación sugerida

* A: alta prioridad.
* B: prioridad media.
* C: baja prioridad.

### Resultado esperado

Lista priorizada de contrapartes.

---

## Etapa 6: Preparar mensaje de invitación

### Objetivo

Solicitar reuniones de forma clara, profesional y prudente.

### Actividades

1. Redactar mensaje base.
2. Adaptar mensaje por tipo de contraparte.
3. Validar tono y contenido.
4. Confirmar que no se prometan resultados.
5. Incluir propuesta de valor breve.
6. Proponer reunión corta y concreta.
7. Registrar fecha de envío.

### El mensaje debe incluir

* Presentación breve.
* Razón del contacto.
* Relevancia potencial.
* Solicitud de reunión.
* Disponibilidad para compartir información.
* Tono cordial y ejecutivo.

### Restricciones

El mensaje no debe prometer:

* Ventas.
* Distribución.
* Exclusividad.
* Contratos.
* Inversión.
* Representación.
* Aprobación regulatoria.
* Condiciones comerciales no autorizadas.

### Resultado esperado

Mensaje de invitación validado.

---

## Etapa 7: Coordinar agenda B2B

### Objetivo

Organizar reuniones confirmadas con datos completos.

### Actividades

1. Confirmar disponibilidad del cliente.
2. Confirmar disponibilidad de la contraparte.
3. Definir fecha, hora y zona horaria.
4. Confirmar modalidad: presencial, virtual o híbrida.
5. Crear enlace o ubicación.
6. Registrar objetivo de la reunión.
7. Enviar confirmación.
8. Registrar estado de la reunión.

### Campos mínimos de agenda

| Campo            | Descripción                                 |
| ---------------- | ------------------------------------------- |
| Cliente          | Empresa participante                        |
| Contraparte      | Empresa o institución                       |
| Contacto         | Persona asistente                           |
| Cargo            | Puesto                                      |
| Fecha            | Día de reunión                              |
| Hora             | Horario                                     |
| Zona horaria     | Zona correspondiente                        |
| Modalidad        | Virtual, presencial o híbrida               |
| Enlace/ubicación | Link o dirección                            |
| Objetivo         | Propósito de la reunión                     |
| Responsable      | Persona de seguimiento                      |
| Estado           | Confirmada, pendiente, realizada, cancelada |

### Resultado esperado

Agenda B2B organizada y confirmada.

---

## Etapa 8: Preparar briefing previo

### Objetivo

Evitar reuniones improvisadas.

### Actividades

1. Resumir quién es la contraparte.
2. Explicar por qué es relevante.
3. Definir objetivo de la reunión.
4. Preparar preguntas clave.
5. Señalar posibles riesgos.
6. Indicar qué materiales usar.
7. Definir siguiente paso deseado.
8. Compartir briefing con el cliente o participante.

### Estructura del briefing

1. Contraparte.
2. Perfil resumido.
3. Razón del fit.
4. Objetivo de la reunión.
5. Preguntas sugeridas.
6. Información que debe presentarse.
7. Puntos sensibles.
8. Siguiente paso deseado.

### Resultado esperado

Briefing previo por reunión.

---

## Etapa 9: Ejecutar reunión

### Objetivo

Conducir una conversación útil, enfocada y accionable.

### Estructura sugerida

1. Presentación breve.
2. Confirmación del objetivo.
3. Presentación de la oferta o necesidad.
4. Preguntas a la contraparte.
5. Identificación de interés.
6. Identificación de objeciones.
7. Información solicitada.
8. Confirmación de siguiente paso.

### Preguntas útiles

* ¿Este producto o solución encaja con su operación o portafolio?
* ¿Qué tipo de condiciones necesitaría evaluar?
* ¿Qué información adicional requiere?
* ¿Qué objeciones ve?
* ¿Quién sería la persona correcta para continuar?
* ¿Tiene sentido una segunda conversación?
* ¿Qué siguiente paso recomienda?

### Resultado esperado

Reunión estructurada y con información útil.

---

## Etapa 10: Elaborar minuta

### Objetivo

Registrar la reunión de forma trazable.

### Contenido mínimo de minuta

| Campo                  | Descripción                                            |
| ---------------------- | ------------------------------------------------------ |
| Fecha                  | Día de la reunión                                      |
| Participantes          | Personas y empresas presentes                          |
| Objetivo               | Propósito original                                     |
| Resumen                | Principales temas tratados                             |
| Interés detectado      | Alto, medio, bajo o nulo                               |
| Objeciones             | Barreras o dudas                                       |
| Información solicitada | Documentos, precios, muestras, contactos               |
| Próxima acción         | Qué sigue                                              |
| Responsable            | Quién debe hacerlo                                     |
| Fecha sugerida         | Cuándo dar seguimiento                                 |
| Clasificación          | Informativa, útil, oportunidad potencial, no relevante |

### Resultado esperado

Minuta clara y accionable.

---

## Etapa 11: Ejecutar seguimiento

### Objetivo

Convertir la reunión en acción concreta.

### Acciones posibles

* Enviar ficha técnica.
* Enviar catálogo.
* Enviar presentación.
* Enviar precios o rango.
* Enviar muestras.
* Agendar segunda reunión.
* Compartir contacto adicional.
* Solicitar información faltante.
* Validar requisitos.
* Registrar no interés.
* Convertir en oportunidad.

### Regla de seguimiento

El seguimiento debe realizarse lo antes posible y debe estar conectado con lo que realmente se habló en la reunión.

### Resultado esperado

Siguiente acción ejecutada y registrada.

---

## Etapa 12: Convertir reunión en oportunidad

### Objetivo

Distinguir una reunión útil de una oportunidad comercial real.

### Señales de oportunidad

Una reunión puede convertirse en oportunidad si existe al menos una señal clara:

* Solicitud de información adicional.
* Solicitud de precio.
* Solicitud de muestra.
* Solicitud de propuesta.
* Reunión de seguimiento.
* Interés en distribución.
* Interés en alianza.
* Introducción a decisor.
* Solicitud de visita.
* Discusión de condiciones comerciales.
* Validación de volúmenes.
* Interés en piloto.

### Regla operativa

No toda reunión es oportunidad. Una reunión solo debe convertirse en oportunidad cuando exista una acción posterior específica o una señal clara de avance.

### Resultado esperado

Oportunidades reales separadas de reuniones informativas.

---

## Etapa 13: Preparar reporte de resultados

### Objetivo

Presentar el resultado del matchmaking de forma ejecutiva.

### Estructura sugerida

1. Resumen ejecutivo.
2. Objetivo del matchmaking.
3. Perfil del cliente o participantes.
4. Criterios de selección de contrapartes.
5. Agenda ejecutada.
6. Reuniones realizadas.
7. Reuniones útiles.
8. Oportunidades potenciales.
9. Objeciones recurrentes.
10. Seguimientos pendientes.
11. Recomendaciones.
12. Siguiente paso sugerido.
13. Anexos: agenda, minutas, matriz de seguimiento.

### Resultado esperado

Reporte ejecutivo de Business Matchmaking.

---

## 5. Uso de IA en el proceso

La IA puede apoyar en:

* Perfilar clientes y contrapartes.
* Sugerir criterios de fit.
* Clasificar contrapartes A/B/C.
* Redactar mensajes de invitación.
* Preparar briefings previos.
* Generar preguntas para reuniones.
* Resumir notas en minutas.
* Identificar señales de oportunidad.
* Convertir notas en acciones.
* Preparar reportes ejecutivos.
* Detectar reuniones de bajo fit.

---

## 6. Controles humanos obligatorios

Una persona de ARNI debe validar:

1. Que la contraparte exista.
2. Que el contacto sea correcto.
3. Que la reunión tenga un objetivo claro.
4. Que el mensaje de invitación sea adecuado.
5. Que no se prometan ventas, alianzas o contratos.
6. Que los materiales enviados estén autorizados.
7. Que la minuta refleje correctamente la conversación.
8. Que el seguimiento sea oportuno.
9. Que la oportunidad tenga señales reales.
10. Que temas legales, regulatorios, sanitarios, aduanales o contractuales sean revisados por especialistas.

---

## 7. Indicadores del proceso

| Indicador                  | Descripción                                          |
| -------------------------- | ---------------------------------------------------- |
| Contrapartes identificadas | Número total de posibles reuniones                   |
| Contrapartes validadas     | Contrapartes con fit razonable                       |
| Invitaciones enviadas      | Mensajes enviados                                    |
| Reuniones confirmadas      | Reuniones aceptadas                                  |
| Reuniones realizadas       | Reuniones efectivamente ejecutadas                   |
| Reuniones útiles           | Reuniones con información o acción relevante         |
| Oportunidades potenciales  | Reuniones con señal clara de avance                  |
| Seguimientos ejecutados    | Acciones realizadas después de reuniones             |
| Seguimientos pendientes    | Acciones abiertas                                    |
| Tasa de conversión         | Oportunidades potenciales sobre reuniones realizadas |
| Calidad de agenda          | Porcentaje de reuniones A o B sobre total            |

---

## 8. Riesgos frecuentes

* Agendar reuniones sin fit real.
* Priorizar cantidad sobre calidad.
* No preparar al cliente antes de la reunión.
* No definir objetivo de reunión.
* No documentar minutas.
* No dar seguimiento.
* Confundir cortesía con interés comercial.
* Convertir reuniones informativas en oportunidades falsas.
* Enviar información no autorizada.
* Prometer ventas, contratos, alianzas o distribución.
* Depender de IA sin validación humana.

## 10. Regla final

El objetivo de Business Matchmaking no es llenar una agenda de reuniones.

El objetivo es generar conversaciones relevantes, preparadas, documentadas y accionables que ayuden al cliente a avanzar con mayor claridad comercial.
